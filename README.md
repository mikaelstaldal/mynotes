# go-web-template

A template for small, self-hosted web applications. It captures the common
architecture of a Go backend with SQLite storage, a REST API defined in OpenAPI,
and an embedded Preact + TypeScript frontend — all shipped as a single binary.

The repository implements one example resource (`items`) end to end. Replace it
with your own domain.

## Stack

- **Backend:** Go, layered `handler → service → repository → SQLite`
  (`modernc.org/sqlite`, pure Go — no CGO).
- **API:** defined in `openapi.yaml`; Go server stubs generated by
  [ogen](https://ogen.dev/), TypeScript types by
  [openapi-typescript](https://github.com/openapi-ts/openapi-typescript).
- **Frontend:** [Preact](https://preactjs.com/) + TypeScript, compiled by `tsc`
  with no bundler; vendored runtime loaded via import maps.
- **Shared server plumbing:** auth, CSRF, gzip, security headers, and panic
  recovery from
  [`go-server-common`](https://github.com/mikaelstaldal/go-server-common).
- **Single-binary deploy:** the frontend is embedded with `//go:embed`.

## Getting started

Prerequisites on `$PATH`: `go`, `ogen`, `tsc`, `openapi-typescript`,
`golangci-lint`.

```bash
./build.sh            # generate code, compile TS, build, test, lint
./go-web-template     # http://127.0.0.1:8080
```

The SQLite database is created automatically at `data/app.sqlite` on first run.

### Useful flags

| Flag | Default | Purpose |
|------|---------|---------|
| `-addr` | `127.0.0.1` | bind address |
| `-port` | `8080` | HTTP listen port |
| `-data` | `data` | data directory (holds the SQLite file) |
| `-public-url` | — | public base URL for CSRF validation behind a proxy; an `https://` URL also enables `Strict-Transport-Security` |
| `-basic-auth-file` | — | htpasswd file (bcrypt) to enable HTTP basic auth |

## Layout

See [AGENTS.md](AGENTS.md) for the full architecture, conventions, and code-
generation workflow. A summary of the directory tree:

```
main.go             # wiring, routing, middleware, graceful shutdown
openapi.yaml        # API contract — source of truth for both generators
internal/
  api/              # generated ogen stubs (gitignored)
  handler/          # api.Handler implementation + middleware
  service/          # validation, sanitization, business rules
  repository/       # SQLite schema, migrations, queries
  model/            # domain types
  sanitize/         # HTML sanitization
web/ts/             # TypeScript frontend sources
web/static/         # embedded assets (HTML/CSS/vendored preact/compiled JS)
```

## Using this as a template

1. Rename the module in `go.mod` and the binary name in `build.sh` / `.gitignore`.
2. Replace the `items` resource in `openapi.yaml` with your own paths and schemas.
3. Update the layers to match: `model`, `repository` (schema + migrations),
   `service`, `handler`, and the frontend `api/client.ts` + views.
4. Run `./build.sh`.
